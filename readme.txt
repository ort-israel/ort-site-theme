== תאור פונקציונאלי של האתר ==

== דף הבית ==
דף הבית מחולק ל -sections כאשר כל אחד מהם מגדיר אזור בפני עצמו.
את הפונקצינאליות של דף הבית, ניתן למצוא בקובץ home-functions.php
כל אזור מושך את הפוסטים הרלוונטים אליו.
את פרטי הפוסט (תאריך,שם קטגוריה וכדומה) אנו מעברים כמפרמטרים לפונקציה mop_get_posts

== דף קטגוריה ==
קיימות 2 סוגי קטגוריות 1)category-horizontal
2) category-vertical
מבחינה פונקציונאלית ההבדל היחיד בין הקטגוריות הוא שבאופקית לא נמשך תקציר ובמאונך כן מופיע תקציר.

הופעת התצוגה הרצויה תלויה בבחירת שדה מיוחד מתוך דף הקטגוריה.

נוכל לראות פונקציונאליות מורכבת של דף הקטגוריה בגגל התורך בהופעת "מנגנון" חיפוש בקטגוריית "כתבות
בכל אחד מתתי הקטגוריות של כתבות יש להציג את הקטגוריות אחים שהם בעצם הילדים של "כתבות"

סוג נוסף הוא content-link - תצוגה מיוחדת של קטגוריית ארכיון ידעונים וקטגוריית "מן העיתונות"

= עמוד =
ניתנת אופציה בראש העמוד להציג תמונת רקע שהיא בעצם התמונה הראשית של עמוד
וכן נדרש להציג תקציר במידה ויש לעמוד
יתר הפונקציונאליות פשוטה וברורה

=פוסטים=
קיימים 4 סוגי תצורות של פוסטים
1) פוסט מומחים
2) פוסט בתי ספר
3) פוסט פורמט לינק - לצורך מבנה מיוחד לפוסטים השייכים לקטגוריות ידיעונים ומן העיתונות.
4) צורה דיפולטיבית של תצוגת פוסט שוטף שהיא בעצם נקראת באותה של שורה של פוסט פורמט

=דף תוצאת חיפוש=
מציג לא רק פוסטים אלא גם עמודים הנבנים באלמנטור.
ולכן היה צורך בכתיבת תנאי מיוחד למקרה שזה עמוד ולא פוסט רגיל.
כמו כן, היתה דרישה להציג תאריכים לפוסטים הנמשכים מקטגרויית חדשות.

= HEADER =
ההדר מורכב משני חלקים: 1) בר-ניווט 2) בר-ניווט שפות
לוגואים.

= FOOTER =
הפוטר העליון נבנה באלמנטור.
הפוטר התחתון נבנה באלמנטור.
שורת הקרדיט מתוגמת מתוך הקוד.

= שדות מיוחדים =
1) מומחים נמשך ע"י שדה מיוחד הנותן אופציה לבחור מתוך רשימת מומחים אלו מומחים יופיעו בעמוד הספציפי.
2) תאריך + שם מחבר נמשך כשדה מיוחד המופיע אך ורק כאשר מסמנים את סוג הפוסט כ "פוסט-לינק".
